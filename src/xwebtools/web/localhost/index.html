XDS Tools

You are connected to localhost copy of the Public Registry.

Allocate Patient ID for the Public Registry - allocate a Patient ID and feed it to the Document Registry actor via the Patient Identity Feed transaction. The Document Registry will reject submissions carrying Patient IDs not allocated though this tool. Remember there are two copies of the Public Registry, one on the Internet and one on the Connectathon floor. Be sure to allocate the patient ID on the correct copy.

Metadata Tools - a collection of tools for working with XDS metadata and documents.

Test Log Browser - review your submissions to the Public Registry. It senses your IP address and gives you access to the submissions that originated at that IP address. It also allows you to type in other IP addresses and view submissions that originated there.

Syslog Browser - review syslog submissions made to 10.242.1.132:8087 (nist1). The same IP address filtering documented for the Test Log Reader applies to this tool.

XDS Test Document Consumer - this tool is used by Connectathon monitors to retrieve and evaluate XDS Registry and Repository contents.
